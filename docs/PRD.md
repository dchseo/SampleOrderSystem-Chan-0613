# PRD — SampleOrderSystem-Chan-0613 (반도체 시료 생산주문관리 시스템)

이 문서는 과제 PDF(`[CRA_AI] Day3_개인과제_반도체시료관리.pdf`) Chapter 1·2의 기능 명세를
요구사항 문서 형태로 재정리한 것이다. 설계 결정(알고리즘, 데이터 구조, 이식 대상 등)은 다루지
않으며, 그 내용은 [CLAUDE.md](../CLAUDE.md)와 [PLAN.md](PLAN.md)를 참고한다.

## 1. 배경

가상의 반도체 회사 "S-Semi"는 다양한 종류의 반도체 시료(Sample)를 생산하여 연구소, 팹리스
업체, 대학 연구실 등에 납품한다. 주문량이 급증하면서 엑셀/메모장 기반 관리로는 아래 문제가
발생했다.

- 주문 처리 여부를 파악하기 어려움
- 공정 예약 후 완성 시점을 알 수 없음
- 이미 재고가 충분한데도 불필요한 추가 공정이 진행됨

이를 해소하기 위해 시료 등록 → 주문 접수 → 승인/거절 → (필요 시) 생산 → 출고에 이르는 전체
흐름을 관리하는 콘솔 기반 시스템을 구축한다.

## 2. 용어 정의

| 용어 | 의미 |
|---|---|
| 시료 (Sample) | 시스템의 가장 기본이 되는 생산 단위. 고유 ID/이름/평균 생산시간/수율을 갖는다. |
| 주문 (Order) | 고객이 특정 시료를 특정 수량만큼 요청한 것. 상태(Status)를 가진다. |
| 생산 라인 (ProductionLine) | 시료를 하나씩 순차 생산하는 단일 설비 흐름. 부족분만 FIFO로 생산한다. |
| 재고 (Stock) | 시료별로 현재 출고 가능한 수량. |
| 수율 (Yield) | 정상 시료 수 / 총 생산 시료 수 (0~1). |

## 3. 시스템 개요

- 콘솔 기반 애플리케이션이며, 담당자가 메뉴 번호를 직접 입력해 조작한다.
- 생산 라인은 공장에서 시료 하나를 생산하는 단일 설비 흐름이며, 시료를 하나씩 생산한다.
- 주문이 들어온 시료에 대해서만 생산한다 (재고 축적을 위한 선제 생산 없음).

### 3.1 주문 상태 흐름

| 상태 | 의미 |
|---|---|
| `RESERVED` | 주문 접수 |
| `REJECTED` | 주문 거절 (정상 흐름 밖, 모니터링 집계 제외) |
| `PRODUCING` | 주문 승인 완료 + 재고 부족으로 생산 중 |
| `CONFIRMED` | 주문 승인 완료 + 출고 대기 중 |
| `RELEASED` | 출고 완료 (PDF 원문 표기는 "RELEASE", 상태 이름은 `RELEASED`로 통일 — CLAUDE.md 참고) |

```
RESERVED --승인(재고 충분)--> CONFIRMED --출고--> RELEASED
RESERVED --승인(재고 부족)--> PRODUCING --생산 완료--> CONFIRMED --출고--> RELEASED
RESERVED --거절--> REJECTED
```

## 4. 기능 요구사항

### 4.1 메인 메뉴

기능별 선택 화면을 표시하고, 전체 시료에 대한 요약 정보(등록 시료 종수, 총 재고, 전체 주문
건수, 생산 라인 대기 건수 등)를 함께 보여준다. 메뉴는 아래 6개 항목 + 종료로 구성한다
(CLAUDE.md "시스템 규칙" 참고 — PDF 기능 명세 표는 5개 카테고리로 설명하지만 예시 UI는 6개
메뉴로 분리되어 있음).

| # | 메뉴 | 설명 |
|---|---|---|
| 1 | 시료 관리 | 신규 시료 등록, 목록 조회, 이름 검색 |
| 2 | 시료 주문 | 고객 주문 접수(예약) |
| 3 | 주문 승인/거절 | 접수된 주문에 대한 승인 또는 거절 |
| 4 | 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 5 | 생산 라인 조회 | 현재 생산 중인 시료 및 대기 큐 확인 |
| 6 | 출고 처리 | `CONFIRMED` 상태 주문에 대한 출고 실행 |
| 0 | 종료 | 프로그램 종료 |

화면 구성 자체는 PDF도 "자유롭게 결정"이라고 명시하므로 레이아웃은 예시 UI를 참고만 한다.

### 4.2 시료 관리

시료는 고유한 이름과 속성을 가지며, 시스템에 등록된 시료만 주문 가능하다.

| 기능 | 입력 | 출력/동작 |
|---|---|---|
| 시료 등록 | 시료 ID, 이름, 평균 생산시간, 수율 | 신규 시료 추가. **재고는 입력받지 않으며 항상 0에서 시작**(CLAUDE.md 참고) |
| 시료 조회 | 없음 | 등록된 모든 시료 목록 + 현재 재고 수량 표시 |
| 시료 검색 | 검색어(이름 등) | 조건에 맞는 시료 목록 표시 |

### 4.3 시료 주문 (예약)

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다.

| 입력 | 설명 |
|---|---|
| 시료 ID | 주문 대상 시료 (등록되지 않은 ID는 거부) |
| 고객명 | 문자열 |
| 주문 수량 | 양의 정수 |

생성 직후 상태는 `RESERVED`이다.

### 4.4 주문 승인/거절

`RESERVED` 상태 주문 목록을 보여주고, 특정 주문에 대해 승인 또는 거절을 처리한다.

- **승인**: 재고 상황에 따라 자동으로 2가지 방식 중 하나로 처리된다.
  - 재고 충분 → 즉시 `CONFIRMED`
  - 재고 부족 → 생산 라인에 자동 등록, `PRODUCING`
  - 정확한 재고 차감/생산량 계산 규칙은 CLAUDE.md "재고 및 생산 라인 처리 규칙 (상세)" 참고.
- **거절**: 즉시 `REJECTED`로 전환.

### 4.5 모니터링

- **주문량 확인**: 상태별(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASED`) 주문 건수를 확인한다.
  `REJECTED`는 유효한 주문이 아니므로 집계에서 제외한다.
- **재고량 확인**: 시료별 현재 재고 수량과, 여유/부족/고갈 3단계 상태를 확인한다.
  - 여유: 주문 대비 재고 충분
  - 부족: 주문 대비 재고 수량 부족
  - 고갈: 수량이 0
  - 상태 갱신 시점과 정확한 비교 기준은 CLAUDE.md 참고. **PDF 예시 UI의 "잔여율"(%) 게이지 바는
    구현하지 않는다** (CLAUDE.md 참고).

### 4.6 생산 라인

- 주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려해 시료를 생산한다.
  - 실 생산량 = `ceil(부족분 / 수율)`
  - 총 생산 시간 = 평균 생산시간 × 실 생산량 (분 단위, 실제 시간 기준)
  - 생산 완료 시 주문 상태 `PRODUCING → CONFIRMED`
- **생산 현황 표기**: 현재 생산 중인 시료에 대한 정보를 표시한다. 표기 정보 수준은 자유(권장:
  진행률/완료 예정 시각 — CLAUDE.md 참고).
- **대기 주문 확인**: 생산 큐에 대기 중인 작업 목록을 FIFO(선입선출) 순서로 표시한다.
- 생산은 애플리케이션 실행 여부와 무관하게 실제 시간 기준으로 진행된 것으로 간주한다 — 상세
  알고리즘은 CLAUDE.md "재고 및 생산 라인 처리 규칙 (상세) §3" 참고.

### 4.7 출고 처리

재고가 충분해진 `CONFIRMED` 주문에 대해 출고를 처리한다. 특정 주문에 대해 출고를 실행하면
상태가 `RELEASED`로 전환된다.

## 5. 비기능 요구사항

| 항목 | 내용 |
|---|---|
| 아키텍처 | Model / View / Controller 계층 분리 (콘솔 MVC) |

| 외부 의존성 | 없음 (JSON/난수 모두 자체 구현 또는 표준 라이브러리만 사용) |
